Three layers of animation work together: **CSS keyframes**, **Framer Motion presets**, and **component-level micro-interactions**.

### 5.2 The Sun Animation (the centerpiece)

Pure CSS keyframes — **no Framer Motion on the sun itself**. Used on every dark-mode page.

```css
/* Outer drift — sun slowly translates across the section */
@keyframes sunDrift { /* set inside each page's <style> block */ }
.sun-drift     { animation: sunDrift 11s ease-in-out infinite;          will-change: transform; }
.sun-drift-rev { animation: sunDrift 15s ease-in-out infinite reverse;  will-change: transform; }

/* Rays rotate 360° in 38s */
@keyframes sunSpin  { to { transform: rotate(360deg); } }
.sun-spin  { transform-origin: center; animation: sunSpin 38s linear infinite; }

/* Core breathes 1 → 1.08 → 1 every 3.6s */
@keyframes sunPulse { 0%,100% { transform: scale(1); } 50% { transform: scale(1.08); } }
.sun-pulse { transform-origin: center; animation: sunPulse 3.6s ease-in-out infinite; }

/* Halo behind the sun: opacity 0.8→1, scale 1→1.12 every 4.5s */
@keyframes sunGlow { 0%,100% { opacity:.8; transform:scale(1); } 50% { opacity:1; transform:scale(1.12); } }
.sun-glow  { transform-origin: center; animation: sunGlow 4.5s ease-in-out infinite; }
```

**SunMark component structure** (replicated in 6+ files — see §11 for refactor note):

```jsx
function SunMark() {
  return (
    <div className="relative h-full w-full">
      {/* 1. Halo — soft glow behind everything */}
      <div className="sun-glow absolute inset-0 rounded-full blur-xl"
           style={{ background: "radial-gradient(circle, rgba(251,191,36,.6), transparent 70%)" }} />

      {/* 2. Rotating rays container */}
      <div className="sun-spin absolute inset-0 h-full w-full">
        <svg viewBox="0 0 200 200">{SUN_RAYS.map((r, i) => <line key={i} ... />)}</svg>
      </div>

      {/* 3. Pulsing core — radial gradient circle */}
      <div className="sun-pulse absolute inset-0 h-full w-full">
        <svg viewBox="0 0 200 200">
          <defs>
            <radialGradient id="sunCoreGrad" cx="42%" cy="38%" r="65%">
              <stop offset="0%"  stopColor="#fff7d6" />
              <stop offset="40%" stopColor="#fcd34d" />
              <stop offset="100%" stopColor="#f59e0b" />
            </radialGradient>
          </defs>
          <circle cx="100" cy="100" r="50" fill="url(#sunCoreGrad)" />
        </svg>
      </div>
    </div>
  );
}
```

**`SUN_RAYS`** is an array of 12 rays (alternating long/short) drawn from center, creating a classic sunburst.

### 6.3 Two-mode design rationale

- **Light sage (home + room pages)**: airy, browsing-friendly, "outdoor morning" feel. White cards pop against the green page.
- **Dark forest (storytelling pages)**: cinematic, intimate, lets the sun animation read as actual sunlight. Amber-300/amber-400 accents act as the "sun's warm glow" on dark green (`#052e23`).

The split is **deliberate** — browsing is light, reading is dark.

## 7. Component Library

### 7.1 UI Primitives (`app/components/ui/`)

**`Button.tsx`** — polymorphic, three variants:
```ts
type ButtonProps =
  | { as?: "button"; ... } // renders <button>
  | { as: "link"; href: string; ... } // renders <Link> from next/link
  | { as: "a"; href: string; ... }; // renders <a>
```
Variants: `primary` (dark → leaf on hover), `secondary` (outline → filled), `ghost` (text link), `light` (cream surface). All wrapped in `motion.span` for hover lift — never `motion.button` directly (Framer 12 + React 19 prop typing conflict).

**`Container.tsx`** — `<div className="container-luxe">` shell with optional `as` prop.

**`ImagePlaceholder.tsx`** — dashed-frame image slot:
- `<ImagePlaceholder kind="hero" | "lifestyle" | "product" | "gallery" | "editorial" | "feature" | "ambience" />`

## 9. Design Decisions & Conventions

1. **Two-mode design (light sage / dark forest)** — deliberate split, not an accident. The sun animation only appears on dark pages.
2. **Amber-300/400 on dark green = "sunlight"** — these warm accent colors are NOT a separate theme; they are the sun's warm light hitting the dark green forest. This is the unifying metaphor.
3. **All shadows tinted green** (`rgba(13, 34, 24, ...)`) — never pure black — to stay on-theme.
4. **Image placeholders, not blank divs** — every image slot is a dashed-frame `<ImagePlaceholder />` so design TODOs are obvious. The component supports all common aspect ratios.
5. **Mobile-first responsive** — every section uses `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3/4` patterns. Hero changes height per breakpoint (`h-[70vh] sm:h-[80vh] lg:h-[92vh]`).
6. **Polymorphic Button** — `motion.span` wrapper around the actual element (avoids Framer 12 + React 19 motion-button type conflicts).
7. **Sun animation is pure CSS** — not Framer Motion. Easy to add/remove per page, GPU-friendly with `will-change: transform`.
8. **Cards lift on hover** — `card-luxe` base class + `zoom-on-hover` for images. Every interactive surface gives feedback.
9. **Sticky nav with smart routing** — `next/link` for client routes, plain `<a>` for hash links.
10. **No animation library on small details** — pulse, marquee, sun all use CSS keyframes; only hero parallax, page reveals, and stat strips use Framer Motion. Keeps the bundle lean.

## 10. Build & Performance Notes

- **Build time:** ~20–35s first build, near-instant rebuilds (Turbopack).
- **Static pre-render:** All 21 routes pre-rendered as static HTML (or SSG for `/rooms/[slug]`).
- **Bundle:** Tailwind v4 JIT, no unused CSS shipped. Framer Motion ~50KB gzipped, used only on client components.
- **Image optimization:** ⚠️ NOT YET DONE. `public/` has **475 MB of unoptimized JPEGs** (avg 3–15 MB per file). Next/Image is configured but most `<img>` tags and `<video poster>` are direct. This is the single biggest performance win available.
- **Video hero:** `public/hero/hero.mp4` is 14 MB — consider WebM/H.264 dual-format + lazy load for mobile.
